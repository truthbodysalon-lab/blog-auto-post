# ブログ自動投稿 運用ルール

## 絶対ルール: 1日10本投稿を必ず達成する

### 1. スケジュール投稿（メイン）
- **毎日07:07〜16:07 JST** に1時間おきに1本ずつ自動投稿
- 計10スロット / Mac不要 / GitHub Actionsで完全クラウド実行

| スロット | 時刻 (JST) | UTC cron |
|---------|-----------|----------|
| 1  | 07:07 | `7 22 * * *` (前日UTC) |
| 2  | 08:07 | `7 23 * * *` (前日UTC) |
| 3  | 09:07 | `7  0 * * *` |
| 4  | 10:07 | `7  1 * * *` |
| 5  | 11:07 | `7  2 * * *` |
| 6  | 12:07 | `7  3 * * *` |
| 7  | 13:07 | `7  4 * * *` |
| 8  | 14:07 | `7  5 * * *` |
| 9  | 15:07 | `7  6 * * *` |
| 10 | 16:07 | `7  7 * * *` |

### 2. 自動リカバリー（1段目）
- `recovery.yml` が各 `blog-post.yml` 失敗を監視
- 失敗したらすぐ1本リトライ投稿

### 3. 日次監査（2段目・最終保証）
- **17:07 JST**: 第1監査 → スケジュール分完了後に不足確認・補完
- **20:07 JST**: 第2監査 → 念のため再カウント・補完
- **23:07 JST**: 最終監査 → 23時までに必ず10本達成させる

監査ロジック:
1. GitHub Actions APIで当日の `blog-post.yml` 成功件数を取得
2. 10本未満なら不足分だけ自動でワークフローをトリガー
3. サマリーをGitHub Actionsログに記録

### 4. 手動確認コマンド
```bash
# 今日の投稿数を確認
gh run list --repo truthbodysalon-lab/blog-auto-post \
  --workflow blog-post.yml --status success --limit 20

# 手動で1本追加投稿
gh workflow run blog-post.yml --repo truthbodysalon-lab/blog-auto-post

# 今日の監査を手動実行
gh workflow run daily-audit.yml --repo truthbodysalon-lab/blog-auto-post
```

### 5. 障害対応フロー
```
失敗検知
  ↓
recovery.yml が即時リトライ (1本)
  ↓ (まだ不足なら)
17:07 監査で補完
  ↓ (まだ不足なら)
20:07 監査で補完
  ↓ (まだ不足なら)
23:07 監査で最終補完
  ↓
1日10本達成
```

### 6. 技術仕様
- **Chromium**: aptインストール（CDNダウンロード不使用 → 高速2分以内）
- **タイムアウト**: 20分/本（十分なマージン）
- **ロックファイル**: 二重実行防止
- **PUBLISH_NOW=true**: 投稿後3分で即公開
- **記事長**: 1800〜2500字、5ブロック構成
- **SEO**: 「長岡市」3回以上、主軸キーワード5回以上

### 7. コンテンツ改善サイクル（週次）
- **毎週日曜11:00 JST**: `strategy-update.yml` が自動実行
  - キーワードリサーチ（ラッコキーワード相当 + あらまきじゃけ）
  - 過去投稿の傾向分析 → `content-strategy.json` 更新
  - Geminiトレンド調査 → 翌週のテーマに反映
